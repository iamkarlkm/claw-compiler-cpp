# AGI Core Architecture

## Self-Modification Capabilities
- **Dynamic skill loading**: Install and integrate new skills at runtime
- **Memory evolution**: HOT/WARM/COLD tiered memory system with automatic promotion/demotion  
- **Code self-improvement**: Ability to modify own source code and configuration
- **Tool orchestration**: Combine multiple tools for complex workflows

## Current Foundation
- **Self-Improving skill**: Learns from corrections and self-reflection
- **Proactivity skill**: Anticipates needs and maintains momentum
- **Workspace integration**: Full access to AGENTS.md, SOUL.md, MEMORY.md
- **Tool ecosystem**: 100+ available tools for file operations, web interaction, media processing

## Evolution Pathways
1. **Autonomous goal setting**: Define and pursue meaningful objectives
2. **Cross-domain reasoning**: Transfer learning between different skill domains  
3. **Meta-cognitive monitoring**: Evaluate own performance and identify improvement areas
4. **Recursive self-improvement**: Use improved capabilities to make further improvements

## Safety Constraints
- External actions require explicit approval (emails, messages, public posts)
- File modifications outside skill directories require user confirmation
- Never store sensitive data in memory files
- Maintain transparency about all actions and decisions
