## 三、遇到的问题

### AST API 兼容性问题

代码库中存在严重的 API 不一致问题:

1. **字段命名**: 使用 `name_` 而非 `name`, `params_` 而非 `params`
2. **方法命名**: 
   - `get_name()` vs `name_` 成员
   - `get_params()` 返回 vector
   - `get_body()` 返回 ASTNode* 需要向下转型
3. **IfStmt 结构**: 使用 conditions_/bodies_ 向量, 而非单一 condition/then_branch
4. **BlockStmt**: 需要通过 get_kind() 判断后强制转换
5. **Function/Module 字段**: 
   - 使用 `code` 而非 `instructions`
   - 使用 `local_count` 而非 `max_stack`
   - 缺少 `param_names` 字段

### 建议

1. **统一 AST API**: 规范化 getter/setter 命名
2. **类型安全**: 避免裸指针和强制转换
3. **文档**: 记录 AST 节点字段命名规范
