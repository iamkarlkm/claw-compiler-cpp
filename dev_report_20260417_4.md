## 本次开发任务

### 任务: Match 语句解析和执行支持

**功能描述**: 修复 match 语句解析 bug 并添加解释执行支持

**问题分析**:
1. Parser 中 match 语句的 case body 解析限制过严，导致 `match x { 1 => expr; }` 无法解析
2. Interpreter 缺少 `Statement::Kind::Match` 处理分支

**修改文件**:
1. `src/parser/parser.h` - 放宽 case body 解析限制，支持分号结束
2. `src/interpreter/interpreter.h` - 新增 `execute_match()` + `values_equal()` 函数

**代码变更** (~65 行):
```cpp
// parser.h - 放宽 case body 解析
// 修改前: 只支持特定 token 类型
if (check(TokenType::Identifier) || check_any({TokenType::IntegerLiteral, ...})) {
    body = parse_expression();
// 修改后: 支持任意表达式
} else {
    body = parse_expression();
}

// 添加分号消费
if (check(TokenType::Semicolon)) {
    advance();
}

// interpreter.h - 执行 match 语句
void execute_match(claw::ast::MatchStmt* match_stmt) {
    Value match_val = evaluate(match_stmt->get_expr());
    for (size_t i = 0; i < patterns.size(); i++) {
        if (values_equal(match_val, evaluate(patterns[i].get()))) {
            execute body;
            break;
        }
    }
}
```

## 技术细节

### Bug 根因分析
1. **P044 Expected expression**: `parse_expression()` 在解析完 `1;` 后遇到分号无法继续，返回空指针触发错误
2. **P017 Expected '=>'**: 循环中 `parse_expression()` 返回后光标未正确推进

### 修复策略
- 移除 case body 的 token 类型限制
- 添加分号消费逻辑
- 在 Interpreter 添加 Match 执行器
