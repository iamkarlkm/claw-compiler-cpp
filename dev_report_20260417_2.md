## 本次开发任务

### 任务: Tuple Expression 元组表达式解析

**功能描述**: 支持 `(expr1, expr2, ...)` 形式的元组表达式解析

**问题**: 之前 `return (1, 2);` 这类语句无法解析

**修改文件**:
1. `src/ast/ast.h` - 新增 `TupleExpr` 类 + ExpressionKind::Tuple
2. `src/parser/parser.h` - 修改 `parse_primary()` 支持元组解析

**代码变更** (~120 行):
```cpp
// ast.h - 新增元组表达式
class TupleExpr : public Expression {
public:
    TupleExpr(std::vector<std::unique_ptr<Expression>> elements,
              const SourceSpan& span);
    const auto& get_elements() const { return elements_; }
    size_t size() const { return elements_.size(); }
    // ...
};

// parser.h - 元组解析逻辑
if (check(TokenType::LParen)) {
    advance();
    // Check for empty tuple: ()
    if (check(TokenType::RParen)) {
        advance();
        return std::make_unique<ast::TupleExpr>(..., span);
    }
    // Parse first expression
    auto first = parse_expression();
    std::vector<std::unique_ptr<ast::Expression>> elements;
    elements.push_back(std::move(first));
    
    // Check for tuple: (expr, expr, ...)
    while (check(TokenType::Comma)) {
        advance();
        auto next_elem = parse_expression();
        elements.push_back(std::move(next_elem));
    }
    // Single element returns just expression
    if (elements.size() == 1) return std::move(elements[0]);
    return std::make_unique<ast::TupleExpr>(std::move(elements), span);
}
```

## 技术细节

### 遇到的问题及修复
1. **AST 克隆接口冲突**: 原始 clone() 返回类型不匹配
   - 修复: 使用 `std::unique_ptr<ASTNode>` 虚函数默认实现
   
2. **编译错误**: 抽象类无法实例化
   - 修复: 为 ASTNode 提供默认 clone() 实现 (返回 nullptr)

### 架构影响
- 新增 `Expression::Kind::Tuple` 枚举值
- Parser 支持完整的 `(expr, expr, ...)` 解析
- 支持尾随逗号 `(1, 2,)` 和空元组 `()`
